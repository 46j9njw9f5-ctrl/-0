# データソース連携（実労働データ化）

将来性・生産性・株関連は Wikidata（無認証）で実データ化済み。
**労働環境（残業・離職率・有給・平均年収など）** は、以下の公的データを
トークン連携することで実名企業にも表示できるようにする受け皿。

| ソース | 取得できる主なデータ | 認証 | 状態 |
| --- | --- | --- | --- |
| 女性活躍・両立支援DB（厚労省） | 月平均残業・有給取得率・女性管理職比率 | **不要（公開CSV）** | **連携済み** |
| しょくばらぼ（厚労省） | 平均残業時間・有給取得率・離職率 | 一括CSV（無償） | 受け皿あり |
| gBizINFO（経産省） | 従業員数・平均勤続年数・女性管理職比率・育休取得率 | 無料トークン | 受け皿あり |
| EDINET（金融庁） | 平均年間給与・平均勤続年数・平均年齢（上場） | 無料APIキー | 受け皿あり |

## 使い方

### 1. 働きやすさ（トークン不要・連携済み）

```bash
node scripts/enrich-positive-db.mjs
```

厚労省 女性活躍・両立支援DB のオープンデータCSVを47都道府県分ダウンロードし、
**法人番号**（Wikidata P3225）で突合して、各社に `laborReal`
（月平均残業・有給取得率・女性管理職比率）を追記する。これで実名企業に
実データの「働きやすさ」が表示される。

### 2. ブラック度・年収など（追加連携）

```bash
cp .env.example .env      # 取得したトークン/CSVパスを記入
node scripts/enrich-labor.mjs
```

設定されたソースの分だけ `metrics`（離職率・残業代等を含む労働指標）・
`avgAnnualSalary` を追記する。未設定なら安全に no-op。

## 設計方針

- **実名企業に推測値を付与しない**。実データが得られた企業だけ労働評価を有効化する。
- 各ソースはアダプタ（`gbizinfo.ts` / `shokubalabo.ts` / `edinet.ts` 相当のロジック）として
  `scripts/enrich-labor.mjs` に実装。フィールドのマッピングは各 API/CSV の仕様に従う。
- しょくばらぼは残業/有給/離職を持つため、ブラック度・働きやすさの中核データになる。
- 労基署「労働基準関係法令違反に係る公表事案」を将来取り込み、証拠ベースの警告に用いる。
